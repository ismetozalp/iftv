# In-flight TV — Plan: EPG (TV Guide)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** External-XMLTV EPG matched to live channels by normalized name → now/next on cards + player strip + per-channel schedule + a channel×time guide grid. Spec: `docs/superpowers/specs/2026-07-06-epg-design.md`.

**Architecture:** Pure `core/epg/` (parse XMLTV, normalize names, build index, now/next). `adapters/cockpitEpg.ts` fetch (curl|gunzip). `stores/epg.ts` (cache/TTL/refresh/getters). `epgUrl` setting. UI: card line, player strip, schedule popover, `GuideView`.

**Tech Stack:** unchanged (Vue3/Vite/TS/Bootstrap/Pinia/Vitest).

## Global Constraints
- Branch `feat/epg` (off `main`). `src/core/**` pure/DI (no DOM/cockpit); cockpit only in adapters. TDD; per-task commit; merge to `main`.
- EPG fetch is plain HTTP (curl), NOT a stream — but still sequential/cached (TTL 12h); never blocks playback. Fetch failure keeps the last cache.
- Match by **normalized display-name** (panel `epg_channel_id` ≠ XMLTV id). Timestamps → absolute epoch ms (honor `±ZZZZ`).
- Adapter shell passes the URL as `$0` (never interpolated) — it's user-settable.

```ts
import cockpit from 'cockpit'
export async function fetchEpgXml(url: string): Promise<string> {
  return (await cockpit.spawn(['sh','-c','curl -fsSL --max-time 60 "$0" | gunzip -c 2>/dev/null || curl -fsSL --max-time 60 "$0"', url], { err: 'message' })) as unknown as string
}
```
